femdata_stnd_FD

Calculates frequency domain photon fluence rate for the 'standard' type FEM mesh in NIRFAST format. This function generates a sparse matrices representing the FEM problem (the time-resolved 'mass' mastrix and sparse sources vectors). Then, it calls the frequency domain solver.

Syntax

[DATA] = femdata_stnd_FD(MESH, FREQUENCY)

Description

[DATA] = femdata_stnd_FD(MESH, FREQUENCY)
MESH is the NIRFAST mesh structure, FREQUENCY is sources frequency (>=0) in Hz.
DATA structure has following entries:
  • DATA.phi - n by m matrix of photon fluence rate at n nodes for m sources. Expressed in .
  • DATA.complex - p elements vector of photon fluence rate for p source-detectors pairs as specified in MESH.link. This vector has COMPLEX entries. Expressed in .
  • DATA.link - copy of MESH.link showing how sources and detectors are linked together into pairs.
  • DATA.amplitude - absolute value of the DATA.complex. Expressed in .
  • DATA.phase - angle (polar coordinate) of DATA.complex. Expressed in degrees.
  • DATA.paa - two colums matrix [DATA.amplitude DATA.phase];
WARNING: if the MESH.link specifies that a given source-detecor pair is OFF, the DATA values at that pair will be NaN.

Examples

%% load mesh
mesh = load_mesh('cylinder_stnd')

mesh = struct with fields:
name: 'cylinder_stnd' nodes: [12695×3 double] bndvtx: [12695×1 double] type: 'stnd' mua: [12695×1 double] kappa: [12695×1 double] ri: [12695×1 double] mus: [12695×1 double] elements: [63810×4 double] dimension: 3 region: [12695×1 double] source: [1×1 struct] meas: [1×1 struct] link: [240×3 double] c: [12695×1 double] ksi: [12695×1 double] element_area: [63810×1 double] support: [12695×1 double]
figure()
% plot boundary nodes
plot3(mesh.nodes(logical(mesh.bndvtx),1),mesh.nodes(logical(mesh.bndvtx),2),mesh.nodes(logical(mesh.bndvtx),3),'c.');
hold on
% plot sources
plot3(mesh.source.coord(:,1),mesh.source.coord(:,2),mesh.source.coord(:,3),'rx','LineWidth',2,'MarkerSize',8);
% plot detectors
plot3(mesh.meas.coord(:,1), mesh.meas.coord(:,2),mesh.meas.coord(:,3),'bo','LineWidth',2,'MarkerSize',8);
%% run time resolved calculation with even time spacing, take the raw field as well
tic; [data] = femdata_stnd_FD(mesh,100e6); toc; data

data = struct with fields:
phi: [12695×16 double] complex: [240×1 double] link: [240×3 double] amplitude: [240×1 double] phase: [240×1 double] paa: [240×2 double]
% get all source-detector distances
rsd = sqrt(sum((mesh.source.coord(mesh.link(:,1),:) - mesh.meas.coord(mesh.link(:,2),:)).^2,2));
figure()
subplot(1,2,1)
semilogy(rsd, data.amplitude, '.')
xlabel('source-detector distance /mm','Interpreter','latex','FontSize',18)
ylabel('photon fluence rate /mm$^{-2}$s$^{-1}$','Interpreter','latex','FontSize',18)
subplot(1,2,2)
plot(rsd, data.phase, '.')
ylim([0 180])
xlabel('source-detector distance /mm','Interpreter','latex','FontSize',18)
ylabel('phase /deg','Interpreter','latex','FontSize',18)
%% plot raw photon fluence rate
% show amplitude coming from sources 1 and 4, combined for visualization
index_source = [1 4];
plotimage(mesh, log10(abs(sum(data.phi(:,index_source),2))));
set(gca,'CLim',[-12 0], 'CameraViewAngle', 7, 'View', [34, 23])
% visualize phase shifts of sources 1 and 4, combined for visualization
phase = angle(sum(data.phi(:,index_source),2));
% shift by 2*pi if negative
phase(phase<0) = phase(phase<0) + 2*pi;
% convert to degrees
phase = phase * 180/pi;
plotimage(mesh, phase);
set(gca,'CLim',[0 180], 'CameraViewAngle', 7, 'View', [34, 23])
